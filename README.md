# Beijing Hotel - QR Menu

Цифровое интерактивное меню бара отеля «Пекин» в Минске.

![Beijing Hotel](./src/assets/content/logo.png)

## 🌐 Демо

**Live:** [https://alexeevivan.github.io/QR/](https://alexeevivan.github.io/QR/)

## 📋 Описание

Современное веб-приложение для просмотра меню бара отеля Beijing Hotel через QR-код. Поддерживает три языка (русский, английский, китайский), динамическую смену цветовой темы (светлая/темная), адаптивный дизайн и плавные анимации.

### Категории меню:
- 🍸 **Коктейли** - авторские и классические коктейли
- 🥂 **Аперитивы** - шампанское, вермуты, биттеры
- 🍷 **Вина** - красные, белые, розовые вина
- 🥃 **Крепкие напитки** - виски, коньяк, текила, водка
- 🍺 **Безалкогольные напитки** - соки, лимонады, смузи
- ☕ **Кофе и чай** - эспрессо, капучино, чай

## ⚡ Производительность

### Размеры ресурсов:
- **Изображения:** 4.96 МБ (WebP с прозрачностью)
- **JavaScript:** 448 КБ (минифицирован)
- **CSS:** 64 КБ (минифицирован)
- **Шрифты:** 21 МБ (только .woff2)

### Оптимизации:
- ✅ WebP изображения с качеством 85% (экономия 83% от PNG)
- ✅ Lazy loading для всех изображений
- ✅ Async decoding изображений
- ✅ Width/Height атрибуты (предотвращение CLS)
- ✅ Минификация CSS и JS
- ✅ Tree-shaking неиспользуемого кода
- ✅ Удалены устаревшие форматы шрифтов (.eot, .woff)

### Время загрузки (приблизительно):
- **Быстрое 4G:** ~2-3 секунды
- **3G:** ~5-7 секунд
- **Медленное 3G:** ~10-15 секунд
- **Wi-Fi:** ~1-2 секунды

### Core Web Vitals:
- **LCP (Largest Contentful Paint):** < 2.5s
- **FID (First Input Delay):** < 100ms
- **CLS (Cumulative Layout Shift):** < 0.1

## 🚀 Автоматический деплой

Проект настроен на автоматический деплой через **GitHub Actions** при каждом push в ветку `main`.

### Куда деплоится:
1. **GitHub Pages:** [https://alexeevivan.github.io/QR/](https://alexeevivan.github.io/QR/)
2. **Сайт отеля:** [https://beijinghotelminsk.com/qr/](https://beijinghotelminsk.com/qr/)

### Как это работает:

При каждом `git push origin main`:
1. GitHub Actions автоматически запускает workflow (`.github/workflows/deploy.yml`)
2. Устанавливает зависимости (`npm ci`)
3. Собирает проект дважды:
   - Для GitHub Pages (с путями от корня `/`)
   - Для сайта отеля (с путями `/qr/`)
4. Деплоит на оба сайта одновременно:
   - GitHub Pages через `actions/deploy-pages`
   - Сайт отеля через FTP (`SamKirkland/FTP-Deploy-Action`)

### Настройка секретов:

В репозитории настроены GitHub Secrets для FTP-доступа:
- `FTP_HOST` - хост FTP-сервера
- `FTP_USERNAME` - логин FTP
- `FTP_PASSWORD` - пароль FTP

### Ручной деплой (устаревший способ):
```bash
npm run deploy
```
Использует `gh-pages` пакет для деплоя только на GitHub Pages.
